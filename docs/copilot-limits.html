GitHub Copilot — Limits & Constraints

Reference for Automatron integration · April 2026

⚠️
Key constraint: The Copilot coding agent (the one that opens PRs autonomously) cannot be triggered via the public GitHub REST or GraphQL API. It can only be assigned through the GitHub web UI. Automatron works around this by creating issues and linking you directly to the GitHub assignment flow.

API Limitations

🚫
No public API for agent assignment. The POST /repos/{owner}/{repo}/issues/{number}/assignees endpoint returns 422 when "copilot" is passed as an assignee. The @github-copilot comment trigger posts successfully (201) but the agent does not activate — GitHub uses an internal, undocumented API for this.
API Action Works? Notes
Create issue via REST API ✓ Yes Used by Automatron to create tasks
Add label / milestone via API ✓ Yes Fully supported
Assign copilot as issue assignee ✗ 422 GitHub rejects it — not a real account slug
Assign copilot-swe-agent[bot] as assignee ✓ Works Correct bot slug — Automatron uses this. Requires Copilot Enterprise or Pro+.

Register repo webhook via API ✓ Yes Automatron auto-registers on project start
Read PR diff via API ✓ Yes Used for AI code review
Merge PR via API ✓ Yes Requires write access
List / read Copilot agent sessions ✗ No No public endpoint
Cancel a running agent task ✗ No Must use GitHub web UI

How Automatron Works Around This

  • 1
    Automatron creates GitHub Issues via REST API Each task from the architect plan becomes a labelled, milestoned issue
  • 2
    Each issue includes an "Assign on GitHub" button in Automatron UI Links directly to the issue on github.com where you click "Assign to Agent"
  • 3
    Copilot agent opens a PR with Closes #N in the body This is the standard convention Copilot follows automatically
  • 4
    GitHub webhook fires → Automatron receives pull_request event Webhook is auto-registered when you set AUTOMATRON_PUBLIC_URL in .env
  • 5
    Issue status updates to PR open — AI review triggers automatically UI board updates in real-time via WebSocket, no manual sync needed
  • 6
    When PR is merged, webhook fires again → status becomes Merged Full lifecycle tracked without any polling

Minimum Required Plan for Automatron

ℹ️
For org repos (recommended): GitHub Copilot Enterprise ($39/seat/month) — enables the coding agent button on issues in org repositories. 300 tasks/seat/month.
✅
For personal repos: GitHub Copilot Pro+ ($19/month) — includes coding agent for personal account repositories. Limited to 10 tasks/month.
🚫
Copilot Business ($19/seat/month) does NOT include the coding agent. The "Assign to Agent" button will not appear on issues, even though chat and completions work fine.

Webhook Event Automatron Listens To

Only the pull_request event is needed. Automatron ignores all other event types.

// POST /api/webhooks/github // X-GitHub-Event: pull_request { "action": "opened", // or "closed" "pull_request": { "number": 42, "html_url": "https://github.com/org/repo/pull/42", "body": "Closes #7", // ← Automatron scans for this "merged": false }, "repository": { "name": "my-project", "owner": { "login": "Quitcode-Dev" } } }
