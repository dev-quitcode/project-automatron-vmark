Meet the AI Team

🧠

Architect

Plans the project, breaks it into tasks, chooses the tech stack, writes documentation, and re-plans when the builder gets stuck.

🤖

Coder

Writes code inside a sandboxed Docker container using the Cline CLI. Creates files, installs packages, runs commands — like a developer.

🔍

Reviewer

Checks each task's output, decides if it passed or failed, records learnings, and escalates when the coder can't fix an issue.

1

You Describe Your Idea

The Architect AI turns your description into a full build plan

You type a plain-English description of what you want to build. It can be as short as "Build me an invoice dashboard with Next.js and Tailwind" or a detailed multi-paragraph spec.

The Architect AI reads your description and generates a complete project plan including: a step-by-step build order, technology choices, an execution contract (task graph), and structured documentation — PRD, Architecture, and Stories.

Before anything starts, the system runs preflight checks to make sure Docker is running, API keys are valid, and the selected AI model is available. If anything is missing, you'll see exactly what to fix.

📄 What the Architect produces

  • PLAN.md — ordered task checklist with phases
  • execution_contract.json — machine-readable task graph
  • stack_config.json — framework, DB, styling choices
  • PRD.md — why we're building this, scope, success criteria
  • ARCHITECTURE.md — system design and data flow
  • STORIES.md — epics and user stories

⏸ Your first approval

You review the plan in the UI. You can approve it as-is or request changes — the Architect will revise and ask again.

↓
2

Setting Up the Workspace

A GitHub repo and sandboxed Docker container are created automatically

Once you approve the plan, Automatron creates a GitHub repository with a proper branch structure (main → develop → feature/automatron-build).

It then spins up an isolated Docker container from the golden image — a pre-built environment with Node.js, Python, and the Cline coding CLI installed. Inside this sandbox, it runs an init script to bootstrap your chosen framework (Next.js, Vite, Python, etc.), creates the folder structure, writes config files, and generates the project documentation.

Everything is committed as the initial scaffold and pushed to GitHub. The container persists across all subsequent build tasks, validation, and preview.

📦 Repository

  • GitHub repo created with branch protection
  • Branch structure: main → develop → feature
  • CI/CD workflows pre-configured

🐳 Docker Container

  • Based on automatron/golden:latest
  • Ubuntu 24.04 + Node.js + Python + Cline CLI
  • LLM API keys injected securely
  • Port allocated for later preview
↓
3

Building — One Task at a Time

The core loop: pick a task → code it → verify → review → repeat

This is where the real work happens. The system picks one task at a time from the plan, hands it to the AI Coder, verifies the result with automated checks, and has the AI Reviewer decide what to do next.

Each completed task is marked done in PLAN.md, recorded in LEARNINGS.md, synced to STORIES.md, and committed to the feature branch. The loop continues until every task in the plan is complete.

📋 Pick Task

Selects the next uncompleted task from the execution contract. Resets attempt counters for fresh tasks.

🤖 AI Codes It

Cline CLI writes code inside the container — creating files, installing packages, running builds. Just like a real developer.

✅ Verify

Runs validation commands (e.g. npm run build) to check if the code compiles and tests pass.

🔍 Review

AI Reviewer classifies the result as SUCCESS, BLOCKER, or AMBIGUITY and decides next action.

🔄 Self-Healing: What happens when something fails

The system is designed to fix problems on its own. Issues escalate through four levels, each more powerful than the last. You're only asked to step in as a last resort.

Level 1  Quick Retry

The verification step automatically re-runs the same task up to 2 times. Most transient failures (flaky builds, network timeouts) resolve here.

Level 2  Smart Fix

The AI Coder gets the error context injected into its prompt and tries to fix the issue itself — up to 2 more attempts. This catches import errors, type mismatches, and small logic bugs.

Level 3  Re-plan

The Architect AI is called in to revise the approach — it sees the workspace state, git diff, and failure details. It produces a plan delta with a new strategy. Up to 2 escalations.

Level 4  Ask You

After exhausting all automatic retries, the system freezes and asks for your guidance. You can provide feedback, adjust the plan, or skip the task.

↓
4

Live Preview

Your app runs live — you interact with it before anything ships

Once every task is complete, Automatron starts a live preview server inside the Docker container. The app is accessible via a URL in your browser — you can click around, test features, and verify everything works.

Before the preview launches, the system validates that all required deployment artifacts exist: Dockerfile, environment config, CI/CD workflows, and deploy documentation.

This is your second and final approval. If the preview looks good, you approve it and the code moves to the next stage. If something needs fixing, you can request changes.

🖥 What happens

  • Workspace contract validated (all artifacts present)
  • Preview server started (npm run dev or similar)
  • Readiness probe polls until the app responds
  • Preview URL generated and shown in the UI

⏸ Your second approval

You interact with the live app and decide: approve for deployment or request changes.

↓
5

Ready to Ship

Code is merged and prepared for deployment

After you approve the preview, the feature branch is automatically merged into the develop branch on GitHub. The code is now in a stable, deploy-ready state.

At this point you can review the full git history, check the CI results, and deploy whenever you're ready.

🔀 What happens

  • Feature branch merged into develop
  • All commits preserved in git history
  • Project status set to "ready for deploy"
↓
6

Deploy

One click to go live via GitHub Actions

When you're ready, click Deploy. Automatron merges the develop branch into main and triggers the GitHub Actions CI/CD pipeline. The system monitors the deployment and updates the project status in real time.

The deploy workflow, Dockerfile, and environment config were all generated during the build phase — everything is pre-configured.

🚀 Deployment steps

  • Develop branch merged into main
  • GitHub Actions CI workflow triggered
  • Deploy workflow runs (Docker build + push)
  • Status tracked in real time via the UI
💬

Real-Time Updates

WebSocket events keep the UI in sync throughout the entire process
Event When it fires What you see
architect:message Architect is generating the plan Plan text streams into the UI in real time, token by token
status:update Any stage or progress change Progress bar updates, stage label changes
builder:log Builder finishes a task attempt Task result card appears with output and status
plan:updated Task completed or plan revised PLAN.md refreshes with updated checkboxes
human:required System needs your input Notification appears, action buttons enabled
run:error Orchestrator encounters a crash Error card appears in the build log with details
