What Automatron Does

Automatron is an AI-powered development platform that transforms a product idea into a structured, executable software project — automatically. It reads a product repository, generates a delivery plan using AI, breaks it into tracked tasks, and hands each task to an AI coding agent for implementation. The team reviews and approves at key milestones; Automatron manages everything in between.

The core value: faster delivery, less coordination overhead, and full visibility at every stage — from brief to built.

What Has Been Delivered

Planning Engine
AI reads the product repo and produces a structured plan — architecture, stories, and tasks — in minutes, ready for human approval.
Task Management
25 tasks published automatically across 6 workstreams on GitHub, each with clear specifications and acceptance criteria for the AI builder.
Live Dashboard
Real-time project board showing status of every task, progress per workstream, and a live activity feed of what the system is doing.
AI Code Review
When work is submitted, an AI reviewer automatically checks it against the original spec and posts a structured pass/fail report — no human needed.
Instant Notifications
The system receives instant signals from GitHub when new work is submitted — the dashboard updates in real time without manual refresh.
Activity Feed
Step-by-step visibility into every action — reading the repo, generating tasks, triggering the AI builder, reviewing code — logged in real time.

What Remains

Blocked

Activate the AI Builder

The coding agent — the component that writes the software — requires a feature to be switched on within the GitHub account settings (Copilot coding agent, available on Business and Pro+ plans). This is the single critical dependency. Once activated, all 25 tasks can be handed to the AI builder immediately.

Config

Register the Notification Channel

The real-time notification system is fully built but needs to be pointed at a public URL. This is a configuration step — approximately 30 minutes of work once the hosting environment is decided.

Pending

Production Deployment

No production hosting has been configured. A decision on infrastructure is needed before the platform can be made available outside the development environment. The system itself is deployment-ready.

Recommended Next Steps

1

Confirm GitHub plan tier and enable the coding agent

Verify the account has Copilot Business or Pro+ and enable the coding agent in repository settings. This single action unlocks the entire build phase and unblocks all remaining workstreams.

2

Decide on hosting environment

Even a lightweight cloud instance is sufficient to stand up the production service and register the webhook notification channel. A decision here unblocks the final 30% of the initiative.

3

Run the first autonomous build cycle

Once the agent is active, trigger "Assign to Copilot" from the dashboard. The full loop — 25 tasks to code to AI review to merge — will run without further manual input.
